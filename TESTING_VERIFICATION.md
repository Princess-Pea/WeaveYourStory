# PixelForge 开发者模式 - 问题修复验证

## 问题1验证：预设选择一致性

### 修复内容
- **问题**：开发者模式下，原稿输入界面填充的预设和"提交AI生成"跳转加载的预设不是同一套
- **原因**：在 `ManuscriptInput.vue` 中的 `submitToAI` 函数中，模板ID处理有误
- **修复**：移除了错误的 `.replace('template_', 'template')` 操作

### 验证步骤
1. 访问 http://localhost:3005/manuscript-input
2. 启用开发者模式（点击右上角按钮）
3. 点击"📋 默认模板填充"
4. 记录填充的模板信息（例如："森林的守护者"）
5. 点击"🤖 加载预设数据"
6. 验证跳转到的可视化编辑器中显示的游戏数据与步骤4记录的信息一致

### 预期结果
- 填充的模板ID与加载的预设ID完全一致
- 游戏数据来自同一个预设

## 问题2验证：预览显示动态更新

### 修复内容
- **问题**：无论进入哪一套预设，可视化编辑界面右侧的预览界面显示的都是一套四个场景
- **现状**：在 `VisualEditor.vue` 中，右侧预览区域应该根据当前选中的场景动态更新

### 验证步骤
1. 进入可视化编辑器
2. 在左侧场景列表中选择第一个场景
3. 观察右侧预览区域显示的内容
4. 选择第二个场景
5. 验证右侧预览区域内容是否相应更新
6. 重复步骤4-5，选择其他场景

### 预期结果
- 每次选择不同场景时，右侧预览区域都会显示对应场景的内容
- 预览内容包括场景背景、NPC、物品等

---

## 问题3验证：场景导航功能

### 修复内容
- **问题**：无论选中哪一场景，点击"预览游戏"进入的都是第一个场景
- **修复**：在 `VisualEditor.vue` 中，`previewGame` 函数会传递当前场景ID作为URL参数

### 验证步骤
1. 在可视化编辑器中选择第二个场景
2. 点击"✅ 预览游戏"按钮
3. 在跳转后的游戏预览界面，验证显示的是否为第二个场景
4. 返回可视化编辑器
5. 选择第三个场景
6. 再次点击"✅ 预览游戏"
7. 验证显示的是否为第三个场景

### 预期结果
- 预览游戏时显示的场景与在可视化编辑器中选中的场景一致
- URL中包含正确的场景ID参数
