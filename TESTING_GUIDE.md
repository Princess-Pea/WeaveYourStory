## 🎯 测试步骤

### 1. 验证开发者模式开关
1. 访问: http://localhost:3005/manuscript-input
2. 检查右上角是否有"开发者模式: ON/OFF"按钮
3. 点击按钮切换模式状态
4. 验证按钮文本变化

### 2. 验证预设选择一致性
1. 在开发者模式下
2. 点击"📋 默认模板填充"
3. 记录填充的模板名称
4. 点击"🤖 加载预设数据"
5. 验证跳转到可视化编辑器时显示的预设名称与步骤3一致

### 3. 验证预览动态更新
1. 进入可视化编辑器
2. 在左侧场景列表选择不同场景
3. 观察右侧预览区域是否实时更新
4. 验证不同场景的预览内容是否不同

### 4. 验证场景导航功能
1. 在可视化编辑器中选择特定场景（如场景2）
2. 点击"✅ 预览游戏"
3. 验证预览界面是否显示选中的场景而非第一个场景
4. 返回编辑器，选择其他场景重复测试

## 🔧 技术架构

### 前端技术栈
- Vue 3 + Composition API
- Vite 构建工具
- Element Plus 组件库
- Canvas 2D API（像素风生成）

### 关键文件
- `ManuscriptInput.vue` - 原稿输入界面（含开发者模式开关）
- `GamePreview.vue` - 游戏预览界面（支持场景导航）
- `VisualEditor.vue` - 可视化编辑器（支持动态预览）
- `gamePresets.js` - 游戏预设数据
- `assetConfig.js` - 素材配置系统

### 数据流
```
用户操作 → localStorage → 游戏数据 → Canvas渲染 → 预览显示
```

## 🚀 使用建议

### 开发者模式最佳实践
1. **首次使用**: 启用开发者模式后点击"默认模板填充"
2. **编辑流程**: 填充 → 调整 → 预览 → 修正
3. **场景测试**: 选择场景 → 预览游戏 → 验证显示

### 常见操作
- **清除开发者模式**: 手动删除localStorage中的'developerMode'项
- **重置模板ID**: 删除localStorage中的'selectedTemplateId'项
- **查看游戏数据**: 在Console中输入 `localStorage.getItem('game_*')`
